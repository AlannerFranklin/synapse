# 🧠 Synapse - 模块化 AI Agent 引擎

**Synapse** 是一个从零开始、使用纯 Go 语言打造的轻量级 AI Agent 执行框架。

它的名字来源于生物学中的“突触（Synapse）”——就像神经元之间通过突触传递信号一样，在 Synapse 框架中，复杂的 AI 任务被拆解为多个独立的“专家节点”，节点之间通过“全局状态机”传递上下文与记忆，最终汇聚成强大的智能决策树。

本项目不仅仅是一个代码库，更是一次探索 AI Agent 架构演进的实验。从简单的串行函数调用，到复杂的有向无环图（DAG），再到如今支持“平行宇宙”和“时光回溯”的**多叉决策树引擎**，Synapse 旨在提供一个极简、透明且高度可扩展的 Agent 构建范式。

---

## 🌟 核心特性演进之路

本项目经历了四个阶段的硬核演进，每一步都在突破 Agent 能力的边界：

### 🛠️ Phase 1: 基础架构与 DAG 引擎
- **统一 Schema**: 抽象出极其干净的消息（Message）、状态（State）和工具（Tool）数据结构。
- **大模型无缝接入**: 原生支持 DeepSeek / OpenAI 等兼容 API。
- **滑动窗口记忆**: 实现了轻量级的短期记忆（ShortTermMemory）管理。
- **DAG 执行流**: 支持串行与并行节点，让复杂的 Prompt 工程结构化。

### 🔍 Phase 2: 可观测性与持久化
- **黑匣子追踪 (TraceLog)**: 记录每个节点的思考过程和状态流转，AI 的决策不再是黑盒。
- **结构化输出**: 强制大模型输出 JSON 格式的思考链（Chain of Thought）。
- **终端回放**: 酷炫的 `/trace` 命令，在终端随时“重播” Agent 的内部脑暴过程。
- **长期记忆**: 基础的文件系统持久化，让 Agent 拥有跨会话的长期记忆。

### 🌳 Phase 3: 蓝图树引擎与“时光机”
- **双向多叉树结构**: 引入带有 `Parent` 和 `Children` 指针的树形节点，支持复杂的决策分支。
- **配置即代码 (DSL)**: 通过 `blueprint.json` 动态构建图结构，免去硬编码的烦恼。
- **完美并发与状态隔离**: 基于 Goroutine 实现真正的并行节点执行；利用深度拷贝（`State.Clone()`）实现“记忆分叉”，各分支互不干扰。
- **⏪ 时光回溯**: 支持随时回退到任意历史节点，或修改历史消息以分裂出全新的“平行宇宙”状态分支。

### 🌐 Phase 4: 现代化 WebUI 交互面板
- **内置极简 Web 服务**: 仅用原生 `net/http` 启动后端，零第三方依赖包袱。
- **多模型一键切换**: UI 支持快速切换 DeepSeek、OpenAI、Qwen 或自定义兼容接口。
- **热加载蓝图**: 支持直接拖拽或点击上传 JSON 文件，动态重塑 Agent 结构。
- **可视化决策树引擎**:
  - 左侧实时渲染对话的树状结构。
  - **路径高亮与隔离**：点击任意历史节点，聊天框瞬间回溯并只显示当前时间线的干净上下文。
  - **悬浮透视 (Tooltip)**：鼠标划过节点，即可偷窥当时的完整对话摘要与底层节点执行轨迹。
  - **一键导出**：支持将当前的蓝图结构直接下载到本地。

## 🚀 快速开始

### 1. 启动引擎
确保你已经安装了 Go (>= 1.20)。无需繁琐的依赖安装，直接在终端执行：
```bash
go run main.go
```

### 2. 进入 Web 交互终端 (推荐)
项目启动后，默认在 `8080` 端口开启 Web 服务。
1. 在浏览器中打开 [http://localhost:8080](http://localhost:8080)
2. 在左侧面板选择你的 **模型提供商** 并输入 **API Key**。
3. （可选）上传你自定义的 `blueprint.json` 文件。
4. 在中间的聊天框开始交互！你可以点击左侧的树节点，体验“时光回溯”和修改分支的魔法。

> **提示**: 在终端模式下，你也可以输入 `/revert` 来体验基础的时光回溯，输入 `/trace` 查看执行轨迹。
